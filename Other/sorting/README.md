# Решения задач сортировки

## Что такое сортировка?

Сортировка в программировании — это процесс упорядочивания набора данных по определенному признаку. Чаще всего элементы выстраивают либо по возрастанию (от меньшего к большему), либо по убыванию (от большего к меньшему).
Сортировать можно любые данные, у которых можно сравнить элементы:
Числа: по величине ([5, 2, 9] превращается в [2, 5, 9]).
Строки и текст: по алфавиту (['яблоко', 'банан'] в ['банан', 'яблоко']).
Объекты: по дате (например, посты в соцсетях от новых к старым) или по цене (товары в интернет-магазине).

### Зачем нужны алгоритмы сортировки?
В реальной разработке сортировка используется постоянно. Главные причины:
Быстрый поиск: Компьютеру невероятно тяжело искать элемент в хаотичном массиве. Но если данные отсортированы, включаются сверхбыстрые алгоритмы (например, бинарный поиск), которые находят нужный элемент за доли секунды среди миллионов записей.
Наглядность: Пользователям неудобно смотреть на неупорядоченные списки (каталоги товаров, таблицы лидеров в играх, списки контактов).
Оптимизация: Сортировка — это фундамент для многих других сложных алгоритмов (обработки графики, баз данных, искусственного интеллекта).
Поскольку задачи бывают разными (где-то критична скорость, а где-то — экономия оперативной памяти), программисты придумали десятки разных алгоритмов сортировки: от простых (как сортировка выбором или пузырьковая) до продвинутых (быстрая сортировка, сортировка слиянием, пирамидальная).

### Сортировка перебором

#### 🧠 Логика работы алгоритма
Вместо классической перестановки элементов «на месте», в данном решении используется подход с разделением памяти:
1. Программа принимает на вход исходный неотсортированный массив чисел.
2. В цикле запускается поиск минимального элемента среди оставшихся чисел.
3. Найденное минимальное число извлекается (удаляется) из исходного массива.
4. Это число сразу же добавляется в конец нового, изначально пустого массива.
5. Процесс повторяется до тех пор, пока исходный массив полностью не опустеет.

В результате работы функции возвращается новый, полностью упорядоченный по возрастанию список.

#### 📊 Характеристики решения
* **Временная сложность:** $O(n^2)$ — алгоритм выполняет вложенные проходы для поиска минимума.
* **Пространственная сложность:** $O(n)$ — под отсортированные данные выделяется отдельный список равной длины.

#### 🛠️ Как использовать проект
Код оформлен в виде независимой функции, которую можно импортировать в другие модули или тестировать на собственных массивах чисел (включая списки с отрицательными и повторяющимися значениями).

2. `bubble.py`
### Сортировка пузырьком (Bubble Sort)
Функция `bubble_sort` представляет собой глубоко оптимизированную версию классического алгоритма:
* Попарно сравниваются соседние элементы `numbers[i-1]` и `numbers[i]`, сдвигая большие значения в конец.
* **Оптимизация индексов:** Граница внутреннего цикла автоматически уменьшается на величину `iteration` внешнего цикла. Алгоритм не тратит время на проверку уже отсортированной правой части массива.
* **Оптимизация по времени (Флаг досрочного выхода):** Внедрен маркер обменов `flag`. Если за целый проход не было сделано ни одной перестановки, алгоритм мгновенно завершает работу через `break`. На уже отсортированных данных скорость возрастает до линейной — $O(n)$.
